Linux 网络编程-文件管理

简介:
   通过使用多进程与多线程技术， 启用一个守护进程来接受服务器的消息并记录日志；
   主进程主要负责新用户的接入。通过 epoll 机制来处理用户信息， 并处理简单的如用户登陆，注册，登出等简单事件；    如果接受到用户的请求是传输文件等需要耗时较多的进程时，便将任务交给线程池来管理。 线程池由一个单独的线程来管理， 当接受到新任务时 ，首先检测空闲线程的剩余量，当有空闲线程时才接收任务， 否则向主线程返回繁忙，拒绝任务；
   用户连接信息用链表来维护。 节点中记录用于管理用户所需的 ip 地址，用户验证的文件位置， 用于通话的socket 等信息；
   用户功能主要有：文件的上传与下载，新建与删除文件或文件夹， 重命名等。
